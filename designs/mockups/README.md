# Design Mockup Organization Guidelines

## Overview
When working with design mockups, it is essential to follow a consistent naming convention and organizational structure. This ensures that files are easy to locate and understand for all team members.

## File Naming Conventions
1. **Project Name**: Start with the name of the project or feature.
2. **Version**: Indicate the version of the mockup (e.g., v1, v2).
3. **Description**: Include a brief description of what the mockup represents.
4. **Date**: Use the date in YYYYMMDD format to indicate when the mockup was created or updated.

### Format
`[ProjectName]_[Version]_[Description]_[YYYYMMDD]`

### Example
`Homepage_v1_HeroSection_20240420`  
This file name indicates it is the first version of a homepage mockup focusing on the hero section created on April 20th, 2024.

## Organization Guidelines
- **Directory Structure**: Organize files by feature or project.  
  For example:
  - `designs/mockups/Homepage`
  - `designs/mockups/ProfilePage`

- **Subfolders**: For larger projects, consider using subfolders to categorize mockups further. For example, you might have:
  - `designs/mockups/Homepage/v1`
  - `designs/mockups/Homepage/v2`

- **Documentation**: Keep a README.md in each mockup directory to explain the specific organization for that section.

## Conclusion
By following these guidelines, we can maintain a well-organized repository of design mockups that is easily navigable and understandable for all team members.
